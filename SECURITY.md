# Security Policy

## Reporting a vulnerability

Report vulnerabilities privately through GitHub private vulnerability reporting for `MALIEV-Co-Ltd/Legacy.Maliev.Workflows`. Do not disclose a vulnerability in a public issue, discussion, pull request, commit, or workflow log.

Never include passwords, tokens, private keys, service-account JSON, environment files, connection strings, customer data, or any other secret in a GitHub issue or other public report. If sensitive material has already been exposed, revoke or rotate it immediately and use the private report only to identify the affected file and rule without repeating the secret value.

Include a concise impact description, affected workflow or script, reproduction steps, and a safe proof of concept when available. Maintainers will coordinate validation, remediation, and disclosure through the private report.

## Supported versions

Only the latest commit on the protected default branch is supported. Shared workflow consumers must pin an audited full commit SHA so that updates are explicit and reviewable.
